NGIAB Containerization Architechture

NGIAB is designed as a multi-layered containerized tool that encapsulates the NextGen Framework along with many components relevant to the NWM within a reproducible environment. By leveraging Docker and Singularity, NGIAB mitigates deployment barriers, ensuring accessibility, consistency, and ease of use across local machines, cloud platforms, and HPC environments.

  • Core Framework

    At its core (Layer 1) lies a suite of integrated hydrological modeling components and hydrofabric, designed to work together within the NextGen Framework.

  • CI/CD Pipeline

    This core is wrapped by the CI/CD Pipeline layer (Layer 2), which leverages GitHub Actions to ensure automated testing, integration, and deployment capabilities for reproducible workflows.

  • NGIAB Containerization

    The NGIAB Containerization layer (Layer 3) provides the containerized environment and essential configuration tools.

  • Technologies and Methods

    The outermost layer (Layer 4), Technologies & Methods, provides broader infrastructure, best practices, and support for deployment across different computing environments (local, cloud, HPC), and facilitates community engagement and contribution.

NGIAB-DataPreprocess

Data Preparation

The Data Preprocess tool simplifies and accelerates the data preparation for NextGen simulations. Data Preprocess tool subsets hydrofabric data, calculates forcings using weighted means of AORC 1km gridded data or NWM retrospective v3 forcing data, and creates configuration files like realization.json and troute.yaml for seamless integration with NextGen. It offers a user-friendly graphical user interface (GUI) and a command-line interface (CLI).

With the intuitive GUI, users can easily select catchments and define date ranges using an interactive map, simplifying the subsetting of hydrofabrics, generation of forcings, and creation of default NGIAB realizations. This streamlined approach reduces procedural complexity, making it easier to prepare input data and run simulations efficiently. It also supports users with a command-line interface (CLI) to prepare input data and run simulations. Designed to reduce procedural complexity, Data Preprocess ensures efficient workflows while maintaining flexibility for customization. Whether you're running a single simulation or automating large-scale workflows, this tool empowers you to focus on your research by handling the heavy lifting of data preparation.

NGIAB-CloudInfra

Cloud Deployment

NGIAB-CloudInfra provides a containerized, pre-configured solution using Docker, simplifying the deployment of the NextGen Framework and NWM 4.0 components for local machines and cloud environments (AWS, Azure, etc.). With automated CI/CD pipelines and built-in Basic Model Interface (BMI) support, researchers can bypass complex setup processes and focus on running customized hydrologic models. It is ideal for scalable simulations, collaborative projects, and rapid prototyping. NGIAB-CloudInfra ensures reproducibility, cross-platform compatibility, and seamless integration with cloud workflows.

NGIAB-HPCInfra

HPC Solutions

NGIAB-HPCInfra leverages Singularity containers to deliver efficient, secure, and scalable execution of the NextGen Framework on HPC clusters. NGIAB-HPCInfra eliminates the need for root access while maintaining performance. Designed for multi-user environments, it integrates seamlessly with job schedulers (SLURM, PBS, etc.) and supports NWM 4.0-ready components, allowing large-scale hydrologic modeling with minimal setup. By combining BMI-coupled interoperability with HPC-optimized workflows, NGIAB-HPCInfra empowers researchers to run high-resolution, continental-scale simulations.

NGIAB-TEEHR

Evaluation and Analysis

TEEHR is a Python-based package designed to empower researchers with advanced tools for iterative and exploratory analysis of hydrologic model performance. Built on PySpark for distributed computing, TEEHR delivers scalable data processing capabilities while ensuring data integrity through its robust internal framework.

This versatile tool enables users to define custom metrics, implement bootstrapping, and perform advanced data manipulation through grouping and filtering operations. TEEHR also provides tools to fetch critical datasets, including USGS streamflow data and NWM gridded and point-based retrospective and near real-time forecast data. TEEHR also supports comprehensive model evaluation and uncertainty analysis. TEEHR enables researchers to investigate key drivers such as climate variability, time-period influences, and basin characteristics. Its flexible tools allow for the exploration of a wide range of metrics, including error statistics, skill scores, hydrologic signatures, and uncertainty quantification.

TEEHR can be used independently through Python notebooks or scripts, making it a versatile and accessible framework for researchers. Whether you’re analyzing trends, evaluating models, or exploring new hypotheses, TEEHR provides the tools you need to advance hydrologic research with confidence.

NGIAB-Visualizer

Data Visualization

The Data Visualizer component, built on the Tethys Platform, provides a powerful, web-based environment for geospatial and time series visualization of hydrological data. This tool allows researchers to explore catchments and nexus points in both spatial and temporal contexts, offering interactive map-based displays and dynamic time series plots. It seamlessly integrates with outputs from the Data Preprocess and TEEHR tools, enabling users to visualize tabular metrics, hydrologic signatures, and model performance trends. By facilitating a deeper understanding of key hydrological processes and model outputs, the Data Visualizer empowers researchers to analyze and interpret complex data with ease. Whether you're examining spatial patterns or temporal trends, this component provides the intuitive tools needed to gain actionable insights from your simulations.

NGIAB-DataStream

Workflow Orchestration

The NextGen Research DataStream is a daily, CONUS-wide hydrologic simulations in the AWS cloud with publicly accessible inputs, outputs and configurations. It leverages the NextGen Framework to orchestrate physical and machine learning-based models within the NGIAB Docker environment. The open-source realization file, available on GitHub, allows researchers to propose edits, enabling continuous refinement of simulation accuracy. Complementing this, DataStreamCLI serves as the backend software, offering a powerful command-line interface to automate data collection, model execution, and output management. Designed for efficiency and reproducibility, DataStreamCLI handles complex workflows, version control, and large-scale simulations (>10,000 catchments) with ease. Together, these tools provide a robust, scalable, and community-accessible platform for advancing hydrologic research and prediction.
